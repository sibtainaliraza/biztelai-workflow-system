# AI Workflow Document

## Overview

This project was developed using an AI-assisted engineering workflow to accelerate development, debugging, architecture planning, and deployment.

The goal was not only to build the application functionality, but also to demonstrate practical usage of modern AI tools during full-stack software development.

# Prompting & Debugging Workflow

Development followed an iterative workflow:

1. Implement feature/module
2. Test locally
3. Identify runtime/API issues
4. Use AI assistance for debugging
5. Refactor implementation
6. Re-test functionality
7. Deploy and validate production behavior

Examples of debugging areas:
- Gemini API model/version errors
- API quota handling
- Multer upload handling
- MongoDB aggregation failures
- CORS policy issues
- Production deployment failures
- Environment configuration issues
